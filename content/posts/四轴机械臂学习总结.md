# 从零打造四轴机械臂固件：STM32F103 + FreeRTOS 实战

## 前言

最近用 STM32F103C8T6（Blue Pill）和 FreeRTOS 做了一个四自由度机械臂的嵌入式固件，支持旋钮手动控制、串口坐标命令、逆运动学解算、路径插补和 OLED 显示。把过程中的架构设计和踩坑经验整理成文，希望对做类似项目的朋友有帮助。

## 项目概况

```
MCU:      STM32F103C8T6 (72MHz, 64KB Flash, 20KB RAM)
RTOS:     FreeRTOS (CMSIS-V2 封装)
工具链:   arm-none-eabi-gcc + CMake + Ninja
协议:     MIT
```

四个舵机分别控制底座旋转（270°）、大臂俯仰（180°）、小臂俯仰（180°）和夹爪开合（180°）。用户可以通过 EC11 编码器旋钮手动微调每个舵机，也可以通过串口发送 `(x, y, z)` 坐标让机械臂自动移动到目标位置并执行抓取/释放。

## 为什么选择 FreeRTOS？

最初想用裸机超级循环，但面临三个问题：

1. **旋钮响应要求即时** — 用户拧旋钮时 OLED 必须立即更新
2. **串口命令需要持续解析** — 不能因为刷屏就丢数据
3. **舵机平滑需要固定帧率** — 步进计算不能被长时间阻塞

用 FreeRTOS 两个任务自然解决了冲突：

| 任务 | 优先级 | 栈大小 | 职责 | 周期 |
|------|--------|--------|------|------|
| FirstTask | 8 | 128 words | 旋钮读取 + OLED 绘制 | 10ms osDelay |
| SecondTask | 24 | 128 words | 串口解析 + 逆运动学 + 平滑步进 | 无内部延迟（抢占式调度） |

实践中发现一个经验：**高优先级任务不要用 `osDelay`，靠抢占就够了**。Task2 没有内部阻塞，FreeRTOS 会在 Task1 `osDelay(10)` 期间自动调度 Task2 执行，双方各得其所。

## 硬件接线

```
舵机 A (底座)   PA0  TIM2_CH1  270°
舵机 B (大臂)   PA1  TIM2_CH2  180°
舵机 C (小臂)   PB1  TIM3_CH4  180°
舵机 D (夹爪)   PB8  TIM4_CH3  180°
OLED            PB6(SCL) PB7(SDA)  I2C1
编码器旋钮      PA8(PA9) TIM1 编码器模式
旋钮按键        PB15  上拉输入
USART2          PA2(TX) PA3(RX)  115200
```

**⚠️ 一个血的教训**：舵机电源和 STM32 逻辑电源一定要分开！舵机瞬时电流可达 1-2A，如果共用稳压器，STM32 会频繁复位。共地但不共 VCC，舵机用外置 5V/2A 电源。

## 旋钮控制：伪线性加速

EC11 编码器接 TIM1 编码器模式，硬件自动计数，不需要外部中断。每次 `Knob_Loop()` 读出差值后把计数器复位到中间值。

旋钮控制的亮点是**伪线性加速**：连续同方向旋转时，步长按 1.6 倍递增：

```c
#define MIN_INCREMENT 1
#define MAX_INCREMENT 20
#define ACCELERATION_RATE 1.6
```

拧得越快，步长越大，从 1° 一直加速到 20°/步。短按旋钮切换控制目标：A → B → C → D → 循环。

夹爪（SERVO_D）有特殊逻辑：顺时针 → 上电张开到 135°，逆时针 → 闭合到 75° 后 500ms 自动断电，省电又保护舵机。

## OLED 显示：帧缓冲驱动

0.96 寸 OLED（CH1116 芯片），I2C 接口。驱动采用经典的双缓冲方案：

```
Task1_Main:
  Knob_Loop()          ← 读编码器
  OLED_NewFrame()      ← 清帧缓冲
  绘制 A/B/C 角度      ← 写入缓冲
  绘制 X/Y/Z FK 坐标   ← 正运动学实时计算
  绘制 2 连杆动画      ← 右列简图
  osDelay(10)          ← 10ms 帧间隔
  OLED_ShowFrame()     ← I2C 刷屏
```

左侧显示 A/B/C 角度、X/Y/Z 末端坐标和 G 夹爪状态，当前选中的舵机用反色高亮。右侧画一个简单的二连杆动画，直观反映大臂小臂姿态。

## 串口协议：ASCII 文本命令

下行命令格式（PC → STM32）：

```
G x y z\n     抓取：移动到 (x,y,z) 并闭合夹爪
R x y z\n     释放：移动到 (x,y,z) 并张开夹爪
```

坐标单位毫米，命令字母不区分大小写。解析直接用 `parse_int()` 手写解析器，不用 `sscanf`。

上行反馈（STM32 → PC）：

```
140 200 6       ← 正运动学计算的末端坐标
OK G            ← 动作类型
```

不可达坐标回传 `ERR\n`。

接收用 USART2 DMA 空闲中断，NORMAL 模式每次接收后重新配置，`HAL_UARTEx_RxEventCallback` 中标记 `data_ready`，Task2 主循环轮询处理。

## 逆运动学：几何法求解

给定 `(x, y, z)`，用几何法反解三个舵机角度：

1. 计算水平投影距离 `dist_xy = sqrt(x² + y²)`
2. 扣除腕长和基座高得到腕部相对坐标
3. 余弦定理计算大臂-目标夹角和肘关节夹角

连杆参数：

| 参数 | 值 |
|------|-----|
| 大臂长 | 135 mm |
| 小臂长 | 145 mm |
| 腕长 | 68.5 mm |
| 基座高 | 95.5 mm |
| 腕高 | 20 mm |

角度约束：

- **A 轴（底座）**：0–270°，超出则钳位返回不可达
- **B 轴（大臂）**：75–155°，超出则钳位
- **C 轴（小臂）**：动态下限 `c_min = 60° - (B - 90°)`，上限 120°

关键设计：**一次性校验，原子写入**。先算出原始角度，全部检查完限位后再一次性写入目标数组，避免写到一半发现越界。

## 路径插补与平滑运动

### 直线插补

收到目标坐标后，计算起点到终点的 Manhattan 距离，按 3mm 步长划分总步数，初始化 `PathPlanner`：

```c
typedef struct {
    uint8_t  active;
    int      sx, sy, sz;      // 起点
    int      ex, ey, ez;      // 终点
    uint16_t total_steps;
    uint16_t step_idx;
    char     cmd_char;        // 'G' / 'R'
    float    fx, fy, fz;     // 浮点累加坐标
    float    fdx, fdy, fdz;  // 每步浮点增量
} PathPlanner;
```

每帧调用 `path_step_to()` 按浮点增量推进，累加器达到 1.0 时才触发一次逆运动学解算和舵机步进，保证长距离精度不丢失。

### 三轴同步步进

A/B/C 三轴通过 `move_all_axes()` 同时步进，没有分阶段移动。每帧调用 `move_axis_toward_target()` 逐帧逼近目标角度。

### S 形曲线调速

远距离加速到 1.3 倍，接近时平滑减速到 0.4 倍：

```c
剩余角度 < 15°  → smoothstep 减速至 0.4×
剩余角度 > 30°  → 1.3× 加速
其他             → 1.0× 匀速
```

路径整体进度也有独立的 S 形曲线：0-20% 加速 → 20-70% 匀速 → 70-100% 减速。

### 回程间隙补偿

底座（A 轴）齿轮传动有回程间隙。检测到方向切换时触发 40 帧隔帧步进补偿，有效消除空程：

```c
if (dir != last_dir && last_dir != 0) {
    bklash_cnt = 40;  // 40帧补偿
}
if (bklash_cnt > 0) {
    bklash_cnt--;
    if (bklash_cnt % 2 != 0) return;  // 隔帧步进
}
```

## 夹爪状态机

夹爪控制是一个 7 状态的状态机，带 10 秒全局超时保护：

```
串口命令解析
     ↓
  PRE_POSITION → 步进夹爪到预设位
     ↓
  IDLE → 等待路径插补 + 三轴到位
     ↓
  WAIT_ARRIVE → CLOSING_STEP (步进至 80°)
  WAIT_ARRIVE_RELEASE → RELEASE_STEP (步进至 130°)
     ↓
  POWEROFF_DELAY → 300ms 后自动断电 → send_ok()
     ↓
  IDLE
```

自适应步进：距目标 >30° 步 3°，>15° 步 2°，≤15° 步 1°，到位后回退 2° 消除舵机震颤。

关键参数：

| 参数 | 值 |
|------|-----|
| 闭合目标角 | 80° |
| 张开目标角 | 130° |
| 断电延迟 | 300 ms |
| 全局超时 | 10000 ms |
| 到位容差 | 1.5° |

## 工程结构

```
3_aix_robotic_arm/
├── APP/           ← 用户应用（task1_main.c / task2_main.c）
├── Lib/           ← 库模块（servo / oled / kk_knob / font）
├── Core/          ← CubeMX 生成（只通过 .ioc 修改）
├── Drivers/       ← HAL/LL 驱动
├── Middlewares/   ← FreeRTOS 源码
├── cmake/         ← 构建配置

```

**黄金规则**：`Core/` 下的文件只在 `.ioc` 中修改后重新生成，不要手改。所有用户代码在 `APP/` 和 `Lib/` 中完成。


推荐用 ST-Link + OpenOCD 烧录，也可以用串口 ISP（需 BOOT0 拉高）。

## 踩坑总结

1. **舵机供电**：STM32 和舵机必须分开供电，共地不共 VCC
2. **OLED 驱动芯片**：CH1116 和 SSD1306 不兼容，初始化序列不同，买屏幕时确认驱动芯片
3. **DMA 半传输中断**：默认使能 HT 中断会造成回调混乱，显式 `__HAL_DMA_DISABLE_IT(huart->hdmarx, DMA_IT_HT)`
4. **浮点精度**：整数坐标 + 浮点累加器比纯整数步进精度高得多，长距离运动尤其明显
5. **夹爪震颤**：到位后回退 2°，消除舵机在极限位置的抖动
