## 二、日常操作

### 2.1 写新文章

```bash
hugo new content posts/我的新文章.md
```

打开生成的文件，填写 Front Matter：

```markdown
---
title: "我的新文章"
date: 2026-06-24
draft: false          # 设为 false 才会发布
tags: ["ROS2", "学习笔记"]
summary: "一句话描述这篇文章讲什么，显示在文章列表卡片上。"
---

正文内容写在这里，支持完整的 Markdown 语法。

## 代码块

```python
print("Hello World")
```

## 列表

- 项目一
- 项目二
```

**Front Matter 必填字段**：

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 文章标题 | `"ROS2 基础入门"` |
| `date` | 发布日期 | `2026-06-24` |
| `tags` | 标签列表 | `["ROS2", "Python"]` |
| `summary` | 文章摘要 | `"本文介绍..."` |
| `draft` | 是否草稿 | `false` |

### 2.2 本地预览

```bash
hugo server
```

浏览器打开 `http://localhost:1313`，修改文件后自动刷新。

### 2.3 发布上线

```bash
hugo
git add -A
git commit -m "新增文章：xxx"
git push origin main
```

推送后 GitHub Actions 会自动构建并部署到 GitHub Pages，通常 1-2 分钟生效。

## 三、站点配置（hugo.toml）

这是最基础的定制入口，改完重新 `hugo` 构建即可。

### 3.1 基本信息

```toml
baseURL = "https://lyyl80.github.io/"   # 你的域名
title = "lyyl80"                          # 站点标题（显示在标题栏和 <title> 中）

[params]
  description = "写代码，也写生活。"       # 站点描述（SEO 用）
  author = "lyyl80"                       # 作者名（显示在页面顶部）
  tagline = "写代码，也写生活。"           # 副标题（显示在作者名下方）
  github = "lyyl80"                       # GitHub 用户名（页脚显示 GitHub 图标链接）
  email = ""                              # 邮箱（填写后页脚显示邮箱图标）
```

**改什么**：换名字、换标语、换 GitHub 链接，都改这里。

### 3.2 导航菜单

```toml
[menu]
  [[menu.main]]
    identifier = "home"       # 唯一标识符（不要重复）
    name = "主页"             # 侧边栏显示的文字
    url = "/"                 # 点击跳转的链接
    weight = 1                # 排序权重，数字越小越靠前（必须 >= 1）

  [[menu.main]]
    identifier = "posts"
    name = "文章"
    url = "/posts/"
    weight = 2
```

**添加新菜单项**：复制一段 `[[menu.main]]`，改 identifier/name/url/weight。注意 weight 必须从 1 开始递增，**不要用 0**（Hugo 会把 0 当作"未设置"排到最后）。

**示例 — 添加一个"友链"页面**：

```toml
  [[menu.main]]
    identifier = "friends"
    name = "友链"
    url = "/friends/"
    weight = 7
```

然后在 `content/friends/_index.md` 创建对应页面：

```markdown
---
title: "友链"
---

## 我的朋友们

- [朋友A的博客](https://example.com) — 全栈开发
- [朋友B的博客](https://example2.com) — 嵌入式开发
```

### 3.3 分页

```toml
[pagination]
  pagerSize = 10    # 每页显示多少篇文章
```

### 3.4 代码高亮

```toml
[markup]
  [markup.highlight]
    style = "dracula"       # 高亮主题（可换成 monokai, github, nord 等）
    lineNos = false         # 是否显示行号
    guessSyntax = true      # 自动猜测代码语言
```

可选主题列表见 https://gohugo.io/content-management/syntax-highlighting/#list-of-chroma-highlighting-styles

## 四、外观设计（CSS 设计系统）

所有视觉样式集中在 `assets/css/style.css`。文件开头定义了一套 **CSS 变量（Design Tokens）**，改颜色/字体/间距只需要改变量值。

### 4.1 颜色系统

```css
:root {
  /* === 浅色主题 === */
  --seed-bg: #F5F0E8;           /* 页面背景色（暖米色） */
  --seed-fg: #1C1917;           /* 主文字颜色（近黑色） */
  --seed-primary: #8E2F2F;      /* 主题色（深红，用于高亮/链接/强调） */
  --seed-accent: #4E7464;       /* 强调色（森林绿，用于标签等） */
  --seed-surface: #EDEAE2;      /* 卡片/侧边栏背景色 */
  --color-gold: #B28A3E;        /* 金色（装饰用） */
  --seed-radius: 4px;           /* 圆角大小 */
}

[data-theme="dark"] {
  /* === 深色主题 === */
  --seed-bg: #1C1917;           /* 深色背景 */
  --seed-fg: #F0EBE1;           /* 浅色文字 */
  --seed-primary: #D4A574;      /* 暖金色主题色 */
  --seed-accent: #7EA893;       /* 鼠尾草绿 */
  --seed-surface: #2D2926;      /* 深色卡片背景 */
  --color-gold: #D4A574;        /* 金色 */
}
```

**换色系示例 — 换成蓝色调**：

```css
:root {
  --seed-primary: #2563EB;      /* 蓝色主题色 */
  --seed-accent: #7C3AED;       /* 紫色强调色 */
}
[data-theme="dark"] {
  --seed-primary: #60A5FA;      /* 浅蓝 */
  --seed-accent: #A78BFA;       /* 浅紫 */
}
```

### 4.2 字体

```css
--font-display: 'Cormorant Garamond', 'Noto Serif SC', ...;  /* 标题字体（衬线体） */
--font-body: 'Source Serif 4', 'Noto Sans SC', ...;           /* 正文字体 */
--font-mono: 'IBM Plex Mono', 'Fira Code', ...;               /* 代码字体 */
```

**换字体步骤**：
1. 在 `layouts/partials/head.html` 中引入新字体的 Google Fonts 链接
2. 在 `style.css` 的 `--font-display` / `--font-body` 中替换字体名

### 4.3 间距与布局

```css
--space-1: 4px;     /* 最小间距 */
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 24px;
--space-6: 32px;
--space-7: 48px;
--space-8: 64px;
--space-9: 96px;    /* 最大间距 */

--sidebar-width: 260px;   /* 侧边栏宽度 */
--max-width: 960px;       /* 内容区最大宽度 */
```

## 五、模板定制

### 5.1 侧边栏 Logo（partials/sidebar.html）

当前是一个黑底金字的 "L" 方块：

```html
<a href="/" class="sidebar-logo" aria-label="Home">L</a>
```

**换成图片**：
```html
<a href="/" class="sidebar-logo" aria-label="Home">
  <img src="/logo.png" alt="Logo" style="width:36px;height:36px;border-radius:4px;">
</a>
```

**换成文字**：直接把 `L` 改成任意字符或缩写。

### 5.2 首页 Dashboard（layouts/index.html）

首页由三个区块组成，你可以自由增删：

```html
<section class="dashboard">
  <div class="dashboard-grid">

    <!-- 区块 1：日历（可删除或替换） -->
    <div class="dash-card dash-calendar">...</div>

    <!-- 区块 2：统计卡片（可改数字来源） -->
    <div class="dash-stats">
      <div class="stat-card">
        <span class="stat-number">{{ len (where .Site.RegularPages "Section" "posts") }}</span>
        <span class="stat-label">篇文章</span>
      </div>
      <!-- 可添加更多 stat-card -->
    </div>

    <!-- 区块 3：最新文章（改 first 4 为 first 6 显示更多） -->
    <div class="dash-card dash-latest">
      {{ range first 4 $posts }}  <!-- 改成 6 就显示 6 篇 -->
      ...
    </div>

  </div>
</section>
```

**改布局列数**：在 CSS 中修改 `.dashboard-grid`：

```css
.dashboard-grid {
  grid-template-columns: 200px 1fr 1fr;  /* 三列：日历200px + 统计自动 + 文章自动 */
  /* 改成两列：grid-template-columns: 1fr 1fr; */
  /* 改成一列：grid-template-columns: 1fr; */
}
```

### 5.3 文章列表页（list.html）

`list.html` 通过 Section 名称判断显示不同内容：

```go
{{ if eq .Section "archives" }}    → 归档时间线
{{ else if eq .Section "tags" }}   → 标签云 + 分组列表
{{ else if eq .Section "posts" }}  → 文章卡片列表
{{ else if eq .Section "search" }} → 搜索页
{{ else if eq .Section "about" }}  → 关于页
{{ else }}                         → 通用列表
```

**添加新的 Section**：
1. 在 `content/` 下创建新目录（如 `content/projects/_index.md`）
2. 在 `list.html` 中添加对应的条件分支
3. 在 `hugo.toml` 的菜单中添加导航项

### 5.4 文章详情页（single.html）

控制单篇文章的显示方式。关键区域：

```html
<!-- 文章标题 -->
<h1>{{ .Title }}</h1>

<!-- 元信息：日期 + 阅读时间 + 标签 -->
<div class="post-meta">
  <time>{{ .Date.Format "2006年1月2日" }}</time>
  <span>{{ .ReadingTime }} min</span>
</div>

<!-- 文章正文（Markdown 渲染后的 HTML） -->
<div class="post-content">{{ .Content }}</div>

<!-- 上一篇 / 下一篇导航 -->
{{ with .PrevInSection }}<a href="{{ .Permalink }}">← {{ .Title }}</a>{{ end }}
{{ with .NextInSection }}<a href="{{ .Permalink }}">{{ .Title }} →</a>{{ end }}
```

**自定义日期格式**：Hugo 使用 Go 的时间格式，参考值 `"2006-01-02 15:04"`。

### 5.5 页脚（partials/footer.html）

显示 GitHub 链接、邮箱、RSS、版权信息。

**添加社交链接**：在 footer.html 中添加新的图标链接，图标可以从 [Feather Icons](https://feathericons.com) 复制 SVG。

### 5.6 页面顶部标题区（partials/profile-header.html）

显示作者名和副标题。内容来自 `hugo.toml` 的 `params.author` 和 `params.tagline`。

## 六、JavaScript 功能（assets/js/main.js）

所有客户端逻辑在一个文件中，无外部依赖。

### 6.1 主题切换

切换深浅色主题，选择保存在 `localStorage`，刷新或跳转后保持。

### 6.2 搜索功能

- 构建时 Hugo 生成 `/index.json`（包含所有文章的标题、URL、内容、标签、日期）
- JS 在客户端获取这个 JSON，做关键词过滤
- 支持 URL 参数 `?q=关键词` 直接搜索

**搜索匹配逻辑**：标题、内容、标签的不区分大小写子串匹配。

### 6.3 首页日历

JS 自动计算当月天数和第一天是星期几，渲染日历网格，高亮今天。

### 6.4 运行天数

```js
var startDate = new Date('2026-06-24');  // 改成你的建站日期
```

### 6.5 快捷键

- `Ctrl+K`：聚焦侧边栏搜索框
- 侧边栏搜索框按 Enter：跳转到搜索页并显示结果

## 七、添加新功能的常见场景

### 7.1 添加一个全新页面

**步骤**：

```bash
# 1. 创建内容文件
mkdir content/projects
# 创建 content/projects/_index.md
```

```markdown
---
title: "项目展示"
---

## 项目一

这是一个很棒的项目。
```

```toml
# 2. 在 hugo.toml 添加菜单项
[[menu.main]]
  identifier = "projects"
  name = "项目"
  url = "/projects/"
  weight = 7
```

```bash
# 3. 构建并发布
hugo && git add -A && git commit -m "添加项目页" && git push
```

### 7.2 修改 Favicon（浏览器标签页图标）

替换 `static/favicon.svg` 的内容。当前是黑底黄 "L"：

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <rect width="32" height="32" rx="4" fill="#1C1917"/>
  <text x="9" y="25" font-family="Georgia, serif" font-size="24" font-weight="bold" fill="#D4A574">L</text>
</svg>
```

换成任意 SVG、emoji、或字母组合。

### 7.3 添加自定义 CSS

如果不想改 `style.css` 主文件，可以在 `assets/css/extended/` 目录下放额外的 CSS 文件（目录已存在但为空）。然后在 `baseof.html` 中引入：

```html
{{ $ext := resources.Get "css/extended/custom.css" | minify | fingerprint }}
<link rel="stylesheet" href="{{ $ext.RelPermalink }}">
```

### 7.4 嵌入视频/图片

文章 Markdown 中直接使用：

```markdown
![图片描述](/images/photo.jpg)

{{< rawhtml >}}
<video controls width="100%">
  <source src="/videos/demo.mp4" type="video/mp4">
</video>
{{< /rawhtml >}}
```

静态文件放在 `static/images/` 或 `static/videos/` 目录下。

## 八、部署与运维

### 8.1 部署流程

```
git push → GitHub Actions 触发 → Hugo 构建 → 上传 public/ → GitHub Pages 部署
```

全程自动，无需手动操作。构建日志在 GitHub 仓库的 Actions 页面查看。

### 8.2 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 改完推送但页面没变 | GitHub Pages CDN 缓存 | Ctrl+Shift+R 强制刷新 |
| 构建失败 | Markdown 语法或模板错误 | 查看 Actions 日志定位报错行 |
| 菜单顺序乱了 | weight 值设了 0 | weight 从 1 开始递增 |
| 主题切换后跳页恢复 | 旧版未加 localStorage | 已修复，刷新即可 |
| 搜索没结果 | index.json 未生成 | 确认 `hugo.toml` 中有 `[outputs] home = ["HTML", "RSS", "JSON"]` |

### 8.3 常用 Hugo 命令

```bash
hugo server              # 本地开发服务器（自动刷新）
hugo server -D           # 本地服务器（包含草稿）
hugo                     # 生产构建 → public/
hugo --minify            # 生产构建 + 压缩 HTML/CSS/JS
hugo new content posts/文章名.md   # 用默认模板创建新文章
```
